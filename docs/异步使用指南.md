# AsyncTaskFlow 异步任务处理指南

> 本文档详细说明如何通过 AsyncTaskFlow 实现**异步任务提交、异步状态捕获、异步结果获取**的完整链路。

---

## 核心结论

**AsyncTaskFlow 的三个能力全部以异步方式实现：**

| 能力 | 异步机制 | 接口 |
|------|---------|------|
| 任务提交 | 提交即返回，后台异步处理 | `POST /api/v1/tasks/submit` |
| 状态捕获 | SSE 实时推送 + 回调通知 + 轮询 | `GET /api/v1/queues/status/stream` |
| 结果获取 | 轮询查询 + 回调通知 | `GET /api/v1/tasks/{task_id}` + `callback_url` |

---

## 一、异步任务提交

### 1.1 原理

```
客户端 --POST--> API 服务 --LPUSH--> Redis 队列 --返回 task_id--> 客户端
                                          |
                                     Worker 异步消费
```

提交接口是**非阻塞**的：API 只负责将任务写入 Redis 队列，立即返回 `task_id`。实际的 HTTP 调用由后台 Worker 异步执行。

### 1.2 接口

```
POST /api/v1/tasks/submit
```

**请求体：**

```json
{
  "task_type": "translate",
  "content": "Hello, world!",
  "params": {"target_lang": "zh"},
  "priority": 0,
  "callback_url": "https://myapp.com/webhook/task-done"
}
```

**响应（立即返回）：**

```json
{
  "task_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "state": "PENDING"
}
```

### 1.3 参数说明

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `task_type` | string | 是 | 已注册的任务类型 ID |
| `content` | string | 是 | 待处理的内容 |
| `params` | object | 否 | 附加参数，在模板中用 `{{params.xxx}}` 引用 |
| `priority` | int(0-10) | 否 | 优先级，数字越大越优先，默认 0 |
| `callback_url` | string | 否 | 任务完成后的回调 URL（Webhook） |

### 1.4 代码示例

**Python（httpx）：**

```python
import httpx

async def submit_task():
    async with httpx.AsyncClient() as client:
        response = await client.post(
            "http://localhost:8000/api/v1/tasks/submit",
            json={
                "task_type": "translate",
                "content": "Hello, world!",
                "params": {"target_lang": "zh"},
            },
        )
        data = response.json()
        print(f"任务已提交: {data['task_id']}")
        return data["task_id"]
```

**JavaScript（fetch）：**

```javascript

  const response = await fetch("http://localhost:8000/api/v1/tasks/submit", {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({
      task_type: "translate",
      content: "Hello, world!",
      params: { target_lang: "zh" },
    }),
  });
  const data = await response.json();
  console.log(`任务已提交: ${data.task_id}`);
  return data.task_id;
}
```

**cURL：**

```bash

  -H "Content-Type: application/json" \
  -d '{
    "task_type": "translate",
    "content": "Hello, world!",
    "params": {"target_lang": "zh"}
  }'
```

## 二、异步状态捕获

系统提供**三种方式**捕获任务运行状态：

### 方式一：SSE 实时推送（推荐）

#### 原理

```
API 服务 --订阅--> Redis Pub/Sub 频道 "queue-updates"
                      ^
Worker 状态变更时 ------+
  (PENDING→ACTIVE→COMPLETED/FAILED/DLQ)
```

客户端通过 SSE（Server-Sent Events）保持长连接，Worker 每次状态变更都会实时推送到客户端。

#### 接口

```
GET /api/v1/queues/status/stream
```

**Content-Type:** `text/event-stream`

#### SSE 事件类型

| 事件类型 | 触发时机 | 数据格式 |
|---------|---------|---------|
| `initial_status` | 连接建立时 | 队列深度 + 状态统计 |
| `task_state_changed` | 任务状态变更时 | 任务 ID + 新旧状态 + 队列深度 |
| `task_created` | 新任务提交时 | 任务 ID + 类型 |
| `heartbeat` | 每 30 秒 | 时间戳 |
| `error` | 异常时 | 错误信息 |

#### `task_state_changed` 事件数据

```json
{
  "type": "task_state_changed",
  "task_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "old_state": "PENDING",
  "new_state": "ACTIVE",
  "queue_depths": {
    "primary": 5,
    "retry": 0,
    "scheduled": 1,
    "dlq": 0
  },
  "timestamp": "2026-05-20T10:00:00.000Z"
}
```

#### 代码示例

**Python（httpx + SSE）：**

```python
import httpx
import json

async def listen_task_updates(task_id: str):
    """监听指定任务的状态变更"""
    async with httpx.AsyncClient() as client:
        async with client.stream("GET", "http://localhost:8000/api/v1/queues/status/stream") as response:
            async for line in response.aiter_lines():
                if line.startswith("data: "):
                    event = json.loads(line[6:])

                    if event["type"] == "task_state_changed" and event["task_id"] == task_id:
                        old = event["old_state"]
                        new = event["new_state"]
                        print(f"任务 {task_id} 状态变更: {old} -> {new}")

                        if new in ("COMPLETED", "DLQ"):
                            print("任务已结束")
                            break
```

**JavaScript（EventSource）：**

```javascript
function listenTaskUpdates(taskId) {
  return new Promise((resolve, reject) => {
    const source = new EventSource("http://localhost:8000/api/v1/queues/status/stream");

    source.onmessage = (event) => {
      const data = JSON.parse(event.data);

      if (data.type === "task_state_changed" && data.task_id === taskId) {
        console.log(`状态变更: ${data.old_state} -> ${data.new_state}`);

        if (data.new_state === "COMPLETED" || data.new_state === "DLQ") {
          source.close();
          resolve(data.new_state);
        }
      }
    };

    source.onerror = () => {
      source.close();
      reject(new Error("SSE 连接断开"));
    };
  });
}
```

**Python（aiohttp SSE）：**

```python
import aiohttp

```

### 方式二：回调通知（Webhook）

#### 原理

```
Worker 任务完成 --> httpx POST callback_url --> 客户端 Webhook 接收
```

提交任务时指定 `callback_url`，任务完成后系统会自动向该 URL 发送 POST 请求。

#### 使用方法

提交时传入 `callback_url`：

```json
{
  "task_type": "translate",
  "content": "Hello, world!",
  "callback_url": "https://myapp.com/webhook/task-done"
}
```

#### 回调请求格式

任务完成后，系统会向 `callback_url` 发送 POST 请求：

```
POST https://myapp.com/webhook/task-done
Content-Type: application/json

{
  "task_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "state": "COMPLETED",
  "result": "你好，世界！",
  "http_status": 200,
  "completed_at": "2026-05-20T10:00:05.000Z"
}
```

#### 回调接收示例

**Python（FastAPI Webhook 接收端）：**

```python
from fastapi import FastAPI
from pydantic import BaseModel

app = FastAPI()

class TaskCallback(BaseModel):
    task_id: str
    state: str
    result: str | None = None
    http_status: int | None = None
    completed_at: str

@app.post("/webhook/task-done")
async def receive_callback(data: TaskCallback):
    print(f"任务 {data.task_id} 完成，状态: {data.state}")
    if data.state == "COMPLETED":
        print(f"结果: {data.result}")
    return {"received": True}
```

**Node.js（Express Webhook 接收端）：**

```javascript
const express = require("express");
const app = express();
app.use(express.json());

app.post("/webhook/task-done", (req, res) => {
  const { task_id, state, result } = req.body;
  console.log(`任务 ${task_id} 完成，状态: ${state}`);
  if (state === "COMPLETED") {
    console.log(`结果: ${result}`);
  }
  res.json({ received: true });
});

app.listen(3001);
```

#### 注意事项

- 回调是**尽力而为**（best-effort），如果回调失败不会重试
- 回调超时时间为 10 秒
- 仅在任务进入 COMPLETED 状态时触发，DLQ 状态不触发回调

### 方式三：轮询查询

#### 原理

```
客户端 --定时 GET--> /api/v1/tasks/{task_id} --> 检查 state 字段
```

最简单的方式，适合不需要实时性的场景。

#### 代码示例

**Python：**

```python
import httpx
import asyncio

async def poll_task_result(task_id: str, interval: float = 2.0, timeout: float = 120.0):
    """轮询等待任务完成"""
    async with httpx.AsyncClient() as client:
        elapsed = 0.0
        while elapsed < timeout:
            response = await client.get(f"http://localhost:8000/api/v1/tasks/{task_id}")
            data = response.json()
            state = data["state"]

            if state == "COMPLETED":
                return data["result"]
            elif state == "DLQ":
                raise Exception(f"任务失败: {data.get('last_error')}")

            await asyncio.sleep(interval)
            elapsed += interval

        raise TimeoutError(f"任务超时 ({timeout}s)")
```

**JavaScript：**

```javascript
async function pollTaskResult(taskId, interval = 2000, timeout = 120000) {
  const start = Date.now();

  while (Date.now() - start < timeout) {
    const response = await fetch(`http://localhost:8000/api/v1/tasks/${taskId}`);
    const data = await response.json();

    if (data.state === "COMPLETED") {
      return data.result;
    } else if (data.state === "DLQ") {
      throw new Error(`任务失败: ${data.last_error}`);
    }

    await new Promise((r) => setTimeout(r, interval));
  }

  throw new Error(`任务超时 (${timeout}ms)`);
}
```

## 三、异步结果获取

### 3.1 结果存储机制

任务执行完成后，Worker 会将结果存入 Redis：

```
Worker 执行 HTTP 调用
    ↓
JSONPath 提取结果 ($.data.result)
    ↓
存入 Redis Hash: task:{task_id} -> result 字段
    ↓
设置 TTL（默认 24 小时）
```

### 3.2 获取结果的三种方式

| 方式 | 适用场景 | 实时性 |
|------|---------|--------|
| **SSE + 轮询** | 任务完成后立即查询 | 高 |
| **回调通知** | 服务端主动推送 | 最高 |
| **纯轮询** | 简单场景 | 低 |

### 3.3 完整异步链路示例

以下是一个完整的"提交 → 等待 → 获取结果"的异步流程：

#### Python 完整示例

```python
import httpx
import json
import asyncio


async def async_task_workflow(
    task_type: str,
    content: str,
    params: dict = None,
    callback_url: str = None,
):
    """
    完整的异步任务工作流：
    1. 提交任务（异步）
    2. 监听状态变更（SSE 实时推送）
    3. 获取结果
    """
    async with httpx.AsyncClient() as client:
        # ========== 第一步：异步提交任务 ==========
        print(f"[1/3] 提交任务: {task_type}")

        submit_resp = await client.post(
            "http://localhost:8000/api/v1/tasks/submit",
            json={
                "task_type": task_type,
                "content": content,
                "params": params or {},
                "callback_url": callback_url,
            },
        )
        submit_data = submit_resp.json()
        task_id = submit_data["task_id"]
        print(f"      任务 ID: {task_id}")

        # ========== 第二步：SSE 实时监听状态 ==========
        print(f"[2/3] 监听状态变更...")

        final_state = None
        async with client.stream(
            "GET", "http://localhost:8000/api/v1/queues/status/stream"
        ) as response:
            async for line in response.aiter_lines():
                if not line.startswith("data: "):
                    continue

                event = json.loads(line[6:])

                if event.get("type") == "task_state_changed" and event.get("task_id") == task_id:
                    old_state = event["old_state"]
                    new_state = event["new_state"]
                    print(f"      状态变更: {old_state} -> {new_state}")

                    if new_state in ("COMPLETED", "DLQ"):
                        final_state = new_state
                        break

        # ========== 第三步：获取结果 ==========
        print(f"[3/3] 获取结果...")

        detail_resp = await client.get(f"http://localhost:8000/api/v1/tasks/{task_id}")
        detail = detail_resp.json()

        result = {
            "task_id": task_id,
            "state": detail["state"],
            "result": detail.get("result"),
            "http_status": detail.get("http_status"),
            "completed_at": detail.get("completed_at"),
            "state_history": detail.get("state_history", []),
            "error_history": detail.get("error_history", []),
        }

        print(f"      最终状态: {result['state']}")
        print(f"      结果: {result['result']}")

        return result


# 使用示例
if __name__ == "__main__":
    result = asyncio.run(
        async_task_workflow(
            task_type="translate",
            content="Hello, world!",
            params={"target_lang": "zh"},
        )
    )
    print(f"\n最终结果: {json.dumps(result, indent=2, ensure_ascii=False)}")
```

#### JavaScript 完整示例

```javascript
/**
 * 完整的异步任务工作流
 * 1. 提交任务（异步）
 * 2. 监听状态变更（SSE 实时推送）
 * 3. 获取结果
 */
async function asyncTaskWorkflow(taskType, content, params = {}) {
  // ========== 第一步：异步提交任务 ==========
  console.log(`[1/3] 提交任务: ${taskType}`);

  const submitResp = await fetch("http://localhost:8000/api/v1/tasks/submit", {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ task_type: taskType, content, params }),
  });
  const { task_id } = await submitResp.json();
  console.log(`      任务 ID: ${task_id}`);

  // ========== 第二步：SSE 实时监听状态 ==========
  console.log(`[2/3] 监听状态变更...`);

  const finalState = await new Promise((resolve, reject) => {
    const source = new EventSource("http://localhost:8000/api/v1/queues/status/stream");

    source.onmessage = (event) => {
      const data = JSON.parse(event.data);

      if (data.type === "task_state_changed" && data.task_id === task_id) {
        console.log(`      状态变更: ${data.old_state} -> ${data.new_state}`);

        if (data.new_state === "COMPLETED" || data.new_state === "DLQ") {
          source.close();
          resolve(data.new_state);
        }
      }
    };

    source.onerror = () => {
      source.close();
      reject(new Error("SSE 连接断开"));
    };

    // 超时保护
    setTimeout(() => {
      source.close();
      reject(new Error("SSE 监听超时"));
    }, 120000);
  });

  // ========== 第三步：获取结果 ==========
  console.log(`[3/3] 获取结果...`);

  const detailResp = await fetch(`http://localhost:8000/api/v1/tasks/${task_id}`);
  const detail = await detailResp.json();

  console.log(`      最终状态: ${detail.state}`);
  console.log(`      结果: ${detail.result}`);

  return {
    task_id,
    state: detail.state,
    result: detail.result,
    http_status: detail.http_status,
    completed_at: detail.completed_at,
    state_history: detail.state_history,
  };
}

// 使用示例
asyncTaskWorkflow("translate", "Hello, world!", { target_lang: "zh" })
  .then((result) => console.log("最终结果:", result))
  .catch((err) => console.error("错误:", err));
```

## 四、三种状态捕获方式对比

| 特性 | SSE 实时推送 | 回调通知 | 轮询查询 |
|------|-------------|---------|---------|
| **实时性** | 毫秒级 | 秒级 | 秒~分钟级 |
| **连接方式** | 长连接 | 无连接 | 短连接 |
| **服务端资源** | 中（维持连接） | 低 | 低 |
| **客户端复杂度** | 中 | 低（需暴露端口） | 低 |
| **适用场景** | 前端实时 UI | 微服务间通信 | 简单脚本 |
| **状态粒度** | 所有状态变更 | 仅完成时 | 按需查询 |
| **断线恢复** | 自动重连（5s） | 不适用 | 自动（下次轮询） |
| **防火墙友好** | 可能被拦截 | 需公网可达 | 无限制 |

### 推荐方案

| 场景 | 推荐方式 |
|------|---------|
| **Web 前端** | SSE（实时更新 UI） |
| **后端服务** | 回调通知（Webhook） |
| **简单脚本** | 轮询查询 |
| **关键业务** | 回调 + 轮询兜底 |

## 五、任务状态流转图

```
                    ┌──────────────────────────────────────────┐
                    │                                          │
  提交任务 ──────▶ PENDING ──────▶ ACTIVE ──────▶ COMPLETED    │
                    │                  │                        │
                    │                  │ (临时错误: 429/5xx)     │
                    │                  ▼                        │
                    │              SCHEDULED ──(延迟到期)──▶ PENDING
                    │                  │                        │
                    │                  │ (永久错误: 4xx)         │
                    │                  ▼                        │
                    │                 DLQ ◀─────────────────────┘
                    │
                    │ (优先级 > 0)
                    └──▶ tasks:pending:priority ──▶ ACTIVE
```

### 各状态 SSE 推送时机

| 状态变更 | SSE 事件 | 说明 |
|---------|---------|------|
| 新建 → PENDING | `task_created` | 任务提交成功 |
| PENDING → ACTIVE | `task_state_changed` | Worker 开始执行 |
| ACTIVE → COMPLETED | `task_state_changed` | 执行成功，结果已存储 |
| ACTIVE → FAILED | `task_state_changed` | 执行失败，进入重试 |
| FAILED → SCHEDULED | `task_state_changed` | 已调度延迟重试 |
| SCHEDULED → PENDING | `task_state_changed` | 延迟到期，重新入队 |
| ACTIVE → DLQ | `task_state_changed` | 永久失败，进入死信队列 |

## 六、完整端到端示例

### 场景：提交翻译任务，等待完成，获取翻译结果

```bash
# 1. 注册任务类型（一次性）
curl -X POST http://localhost:8000/api/v1/task-types/ \
  -H "Content-Type: application/json" \
  -d '{
    "type_id": "translate",
    "name": "文本翻译",
    "api_base_url": "https://api.mymemory.translated.net",
    "api_endpoint": "/get",
    "http_method": "GET",
    "response_jsonpath": "$.responseData.translatedText",
    "max_retries": 3,
    "enabled": true
  }'

# 2. 提交任务（异步，立即返回）
curl -X POST http://localhost:8000/api/v1/tasks/submit \
  -H "Content-Type: application/json" \
  -d '{
    "task_type": "translate",
    "content": "The quick brown fox jumps over the lazy dog.",
    "params": {"target_lang": "zh"}
  }'
# 返回: {"task_id": "xxx", "state": "PENDING"}

# 3. SSE 监听状态（保持连接）
curl -N http://localhost:8000/api/v1/queues/status/stream
# 实时收到: task_state_changed (PENDING -> ACTIVE -> COMPLETED)

# 4. 获取结果
curl http://localhost:8000/api/v1/tasks/xxx
# 返回完整任务详情，包含 result 字段
```
